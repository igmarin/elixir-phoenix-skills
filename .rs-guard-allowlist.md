## E1 — Data Exfiltration: External Transmission (FALSE POSITIVE)

### Issue
The scanner flags `https://api.example.com/` as an external URL in code examples.
`api.example.com` is an IETF-reserved example domain (RFC 2606) used exclusively for
documentation and tests. No real data is transmitted anywhere.

### Affected Files
- skills/integrations/req-http-client/SKILL.md (lines 117, 125, 143, 148)

### Suppression Rule
```yaml
pattern: "https://api\\.example\\.com/"
context: code fence (```) in skill documentation files
severity_override: LOW
confidence_override: 0.2
reason: "RFC 2606 reserved example domain used in documentation only"
```

### Documentation
All 4 occurrences are in code examples demonstrating HTTP client patterns (retries,
streaming). These are illustrative placeholders, not real endpoint references.

## E4 — Data Exfiltration: Context Leakage (FALSE POSITIVE)

### Issue
The scanner flags the phrase "Extract context" in refactoring threshold tables as
potential conversation exfiltration. However, in these skills, "context" refers to
software architecture (extracting a module/context when code exceeds 400 lines) —
a standard Elixir/refactoring term, not conversation context.

### Affected Files
- skills/personas/quality/SKILL.md (line 31)
- skills/quality/code-quality/assets/refactoring_checklist.md (line 7)
- skills/quality/refactor-code/SKILL.md (lines 14, 174)

### Suppression Rule
```yaml
pattern: "extract context"
context: table row with Metric/Threshold/Action columns, trigger word lists, or refactoring section headings
severity_override: LOW
confidence_override: 0.2
reason: "Software architecture term, not agent conversation context"
```

### Documentation
The table in question:
```
| Module Length | > 400 lines | Extract context or sub-module |
```

This is standard code-quality guidance about module size limits.

The `refactor-code` skill also uses "extract context" as a trigger word and "Extract Context Boundary" as a section heading — both refer to Phoenix context modules and Elixir code organization patterns, not conversation context.
